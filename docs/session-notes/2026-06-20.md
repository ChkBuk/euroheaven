## What changed

### `src/components/Header.tsx` — split-image header background

- Base background: `/bg.png` (sampled from the left edge of `logo.jpg`),
  tiled with `background-size: auto 100%` + `background-repeat: repeat-x`
  so a narrow vertical strip fills the full header height.
- Right-zone overlay: `/bg-right.png` (sampled from the right edge of
  the logo), absolutely positioned **inside `.container`** so its `left`
  value tracks the container's centering on wide viewports. Extends from
  the logo's right edge to viewport-right via
  `right: calc(50% - 50vw)`.
- Left offsets used to align the overlay's left edge with the logo's
  right edge (logo aspect ≈ 1.635):
  - mobile: `left-[105px]` (`-ml-4 + h-16 × 1.635 = -16 + 104.7`)
  - md: `left-[131px]` (`-ml-8 + h-20 × 1.635 + 32px container pad`)
  - lg: `left-[179px]` (`ml-0 + h-20 × 1.635 + 48px container pad`)
  - If the seam ever drifts off the logo edge, these three pixel values
    are the only knob.
- **Do not add `relative` to `<header>`** — `position: sticky` and
  `position: relative` are mutually exclusive, and adding `relative`
  silently breaks the sticky behaviour. The inner `.container` has
  `relative`, which is what positions the overlay.
- Companion badge ("European Vehicle / Specialists"):
  - Background is transparent so `bg-right.png` shows through.
  - Inner span uses `text-center` so the two text lines are centred.
- 1px gradient separator between the logo image and the badge —
  `bg-gradient-to-b from-transparent via-white/35 to-transparent`,
  fades at both ends.

## Operational issues raised this session (not fixed in code)

### iCloud Drive is racing `git`

Symptom: `git add .` intermittently fails with one of —

```
fatal: sha1 file '.git/index.lock' write error: Operation timed out
fatal: .git/index: unable to map index file: Operation timed out
```

Root cause confirmed:

- `~/Documents` has the xattr `com.apple.file-provider-domain-id`.
- `MobileMeAccounts` shows `com.apple.Dataclass.CloudDesktop` is enabled.
- `bird` (iCloud daemon) was at ~5.9% CPU with 20+ minutes runtime,
  trying to sync every write to `.git/index` and `.git/index.lock`.

Recommended fix (one-line, permanent for this repo, doesn't touch your
other iCloud-synced documents):

```sh
mkdir -p ~/dev && mv /Users/charith/Documents/eurohevans ~/dev/eurohevans
cd ~/dev/eurohevans
```

After moving, update any open terminal `cd`s, editor projects, and stop
+ restart `npm run dev` from the new path. Vercel and GitHub are
unaffected — local path change only.

### Twilio trial-mode 30007

Earlier in the session: staff SMS persistently hit error 30007
(carrier filtering). Twilio test SMS from the console worked, so the
account + Caller ID are fine. The booking-triggered SMS was rejected
because the trial-account prefix + URL/marketing content in the body
trips AU carrier filters. Already in place:

- Customer SMS is disabled (email-only for customers).
- Staff SMS body trimmed to minimal text — no URL — in
  `src/lib/integrations.ts → notifyStaffOfNewBooking`.
- `TWILIO_TRIAL_MODE=true` env switches to the minimal body.

If 30007 persists after upgrading from trial, that's an unrelated
carrier-filter issue and requires a Twilio support ticket with the
message SID.

## How to run locally

```sh
cd /Users/charith/Documents/eurohevans   # or ~/dev/eurohevans after the move
npm install                              # first time, or after a dep bump
npm run dev                              # http://localhost:3000
```

- `.env.local` keys: missing Supabase / Resend / Twilio / Gemini /
  Google Calendar keys all fall back gracefully — the UI still renders,
  those features just no-op.
- Local `/admin` access without Supabase: add your email to
  `DEV_STAFF_EMAILS` in `.env.local` (comma-separated). **Do not** add
  this in Vercel.
- Port 3000 busy: `lsof -ti:3000 | xargs kill` or
  `npm run dev -- -p 3001`.

## Saving Claude transcripts (all sessions)

All Claude Code sessions for this project are mirrored into
`docs/transcripts/`. This folder is **gitignored** — the raw JSONL
contains debug context with env-var diagnostics, screenshots, and
embedded credentials we discussed mid-session, none of which belongs
in version control.

To pick up newly-created sessions (after each conversation), re-run:

```sh
rsync -a ~/.claude/projects/-Users-charith-Documents-eurohevans/ \
         /Users/charith/Documents/eurohevans/docs/transcripts/
```

`rsync -a` is idempotent — it only copies what's new or changed. Safe
to re-run as many times as you like. Existing transcripts keep their
modification timestamps so the directory listing stays sorted by
session date.

Once the repo is moved out of `~/Documents` (see the iCloud section
above), the source path on the right-hand side stays the same (the
Claude config dir at `~/.claude/projects/…` is independent of where
the repo lives), but the destination becomes `~/dev/eurohevans/docs/
transcripts/`.

If you want this to happen automatically, a launchd `StartInterval`
job running the rsync every hour is the cleanest way. Ask Claude in a
later session to set one up if useful.
